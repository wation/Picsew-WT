# iOS主应用运行选项缺失修复计划

## 问题分析

当前Xcode中主应用的运行选项缺失，可能的原因：

1. **Scheme问题**：
   - 主应用的Scheme可能被误删除
   - Scheme配置可能存在错误
   - Scheme可能没有设置为共享

2. **Xcode缓存问题**：
   - Xcode的缓存可能导致显示异常
   - 项目索引可能损坏

3. **目标配置问题**：
   - 主应用目标的可执行文件设置可能有问题
   - 目标的产品类型可能配置错误

4. **项目文件问题**：
   - 项目文件可能存在损坏
   - 配置可能不完整

## 解决方案

### 1. 检查并重新生成Scheme

**操作步骤**：
1. 打开Xcode项目
2. 点击顶部的Scheme选择器
3. 选择"New Scheme..."
4. 选择主应用目标"Picsew-WT"
5. 点击"OK"
6. 确保新创建的Scheme被设置为当前Scheme

### 2. 清理Xcode缓存

**操作步骤**：
1. 关闭Xcode
2. 删除DerivedData目录：`rm -rf ~/Library/Developer/Xcode/DerivedData/Picsew-WT-*`
3. 清理项目缓存：`xcodebuild clean -project Picsew-WT.xcodeproj -scheme Picsew-WT`
4. 重新打开Xcode

### 3. 验证主应用目标配置

**操作步骤**：
1. 在Xcode中，选择项目名称
2. 选择主应用目标"Picsew-WT"
3. 切换到"General"标签
4. 确保"Product Name"为"Picsew-WT"
5. 确保"Bundle Identifier"为"com.magixun.picsewwt"
6. 确保"Deployment Info"中的"Deployment Target"设置正确
7. 确保"Team"设置正确

### 4. 检查可执行文件设置

**操作步骤**：
1. 切换到"Build Settings"标签
2. 搜索"EXECUTABLE_NAME"
3. 确保值为"$(PRODUCT_NAME)"
4. 搜索"PRODUCT_BUNDLE_IDENTIFIER"
5. 确保值为"com.magixun.picsewwt"

### 5. 验证项目文件完整性

**操作步骤**：
1. 运行`xcodebuild -list`检查项目结构
2. 确认主应用目标存在
3. 确认Scheme存在

## 预期效果

修复后，Xcode应该能够：
1. 显示主应用的运行选项
2. 允许选择主应用作为当前Scheme
3. 能够成功运行主应用
4. 能够在设备上安装和调试主应用
